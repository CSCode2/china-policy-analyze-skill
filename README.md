# China Policy Analyze Skill

中国政策文本是理解中国发展方向的重要公开资料。

很多产业变化、地方行动、监管趋势和资源配置，并不是突然出现的。它们往往先出现在规划纲要、会议通稿、部委文件、地方政策、试点方案和监管文件中。

但政策信息分散、更新频繁、语言专业，普通人很难长期跟踪和准确理解。

本项目希望把这些公开信息整理成一个可追踪、可解释、可验证的知识系统。

它持续关注中央政策、部委文件、地方落地、司法执法、外贸外交、国际经贸规则和外部环境变化，帮助用户理解：

- 政策原文说了什么；
- 政策语言背后代表什么强度；
- 它和过去政策有什么关系；
- 它处在方向提出、试点探索、复制推广还是监管强化阶段；
- 它可能影响哪些行业、地区和人群；
- 哪些判断有依据，哪些结论不能过度解读。

本项目不是官方项目，也不代表任何机构立场。

它不做神秘预测，不提供投资建议，不输出战争时间判断，也不提供任何规避监管、规避制裁或违法套利方案。

它要做的是：让公开政策信息更容易被理解、比较、追踪和验证。

本项目是一个基于公开来源的中国政策理解与分析工具。它用于帮助用户整理、检索、解释和比较公开政策信息，理解政策语言、政策链路、地方落地和外部环境变化。

一句话：**把分散、复杂、专业的公开政策信息，整理成可追踪、可解释、可验证的知识系统。**

## 使用边界

本项目只处理公开来源信息，不收集、推断或传播非公开、涉密、军事部署、个人隐私或敏感身份信息。

本项目不提供：

- 投资建议；
- 法律意见；
- 税务意见；
- 金融建议；
- 军事判断；
- 战争时间预测；
- 制裁规避方案；
- 监管规避方案；
- 出口管制规避方案；
- 违法套利建议。

所有分析仅用于公开信息整理、学习研究、政策理解和风险意识提升。用户应以官方最新文件和专业机构意见为准。

## 为什么做这个项目？

中国政策有一个鲜明特点：**方向一旦写进重要文件，往往不会停留在纸面上。** 它会被部委细化，被地方承接，被**资金、项目、土地、税收、信贷、考核**等机制逐步推动落地。

政策语言也不是随便写的。**"鼓励""支持""大力发展""积极稳妥""规范发展""依法严厉打击"**，这些词背后代表的是不同的政策强度、执行节奏和风险边界。

> **《中国制造2025》就是一个例子。** 2015 年，它提出制造强国的十年行动纲领，重点部署新一代信息技术、高端装备、新能源汽车、新材料等方向。十年后，中国在新能源汽车、光伏、工业机器人、5G 工厂、智能制造等领域已经形成了明显优势。很多后来的产业变化，其实早就写在政策蓝图里。

这说明：**研究中国公开政策，是普通人理解未来中国发展方向的一种高性价比方法。**

它不是唯一方法，也不能保证每个方向都兑现。但它能提前告诉你：

- **国家重视什么**
- **资源流向哪里**
- **哪些行业会被扶持**
- **哪些风险会被约束**
- **哪些规则正在变化**

这是一种**重要的公开信息红利**。

---

问题是，**大多数人没有意识到这一点。** 意识到的人，也很难每天跟踪几十个政府网站，读懂大量公文，理解政策语言，再把中央文件、地方政策、产业动态、司法执法、外贸外交和风险信号整合起来。

**China Policy Analyze Skill 要解决的就是这个问题。**

它持续跟踪权威公开来源，自动采集政策文件、重要讲话、会议通稿、地方落地文件、司法执法案例、外贸外交动态和外部环境风险信号。

然后，它把这些材料翻译成普通人能看懂的分析：

- 这句话是什么意思？
- 政策力度有多大？
- 和过去有什么关系？
- 现在处在试点、推广还是监管阶段？
- 会影响哪些行业和人群？
- 哪些判断有依据，哪些不能过度解读？

```
             - 你是省内短途物流，这不在"公转铁"的重点范围内
             - 公转铁针对的是煤炭、矿石、钢铁这类大宗货，
               走公路成本高、污染大，铁路更划算
             - 短途、零担、门到门——铁路做不了"最后一公里"，
               这恰恰是你们的优势

             关于"多式联运"和"一单制"：
             - 这反而是机会——铁水联运的"最后一公里"和"最初一公里"
               都需要公路短驳
             - "一单制"意味着物流各环节要打通，做短驳的如果信息化水平高，
               反而能更容易接入大物流体系

             ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
             真正要注意的风险
             ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

             1. 环保标准趋严：国六排放标准继续推进，老旧柴油车可能面临
                限行或淘汰——《国务院关于推进服务业扩能提质的意见》
               （国发〔2026〕7号，2026年4月14日）提到"推动老旧设备
                更新改造"，结合新能源车购置税优惠政策（2026年4月2日，
               财政部/税务总局公告），换车成本可能降低
             2. 信息化要求："实施加快数智供应链发展专项行动，推进商贸物流
                数智赋能工程"——小物流公司如果不做信息化，
                可能被平台型物流公司挤掉
             3. 行业整合：政策提到"加快培育综合物流集成商"——
                小公司要么加入大平台的生态，要么做细分领域的"专而小"

             ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
             建议
             ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

             ✅ 不用担心"公转铁"直接淘汰你——你的赛道不在它的目标范围
             ✅ 可以主动对接铁路货站和港口做短驳，这是"公转铁"带来的
                新需求
             ✅ 花点钱做信息化（TMS运输管理系统、电子回单），
                这是对接大平台的前提
             ✅ 关注新能源轻卡替换——政策有补贴窗口，迟早要换

             免责声明：以上为政策文本解读和个人推断，不构成经营建议。
             具体经营决策请结合当地实际情况和专业意见。

             证据等级：E5（政策原文）/ E1（个人推断已标注）
```

## 它不能做什么？

老实说在前头：

- **不给投资建议。** 政策分析一定带着风险提示，政策扶持不等于商业成功。
- **不做确定性预测。** "某年某月一定会发生某事"这种话我们不说，也反对别人说。
- **不碰涉密和军事内容。** 这是红线。
- **不替你做决策。** 工具提供信息，决策永远是你自己的事。
- **不做制裁规避建议。** 观察制裁信息是为了帮你了解风险，不是教你绕开它。

## 自动更新

**本仓库由 Hermes Agent 自动维护。** 以下任务会定时运行，自动采集新政策、更新语料库、生成分析报告，并通过 PR 合入仓库：

| 任务 | 频率 | 做什么 |
|------|------|--------|
| 政策监测 | 每日 06:00 UTC | 采集最新政策文本、生成日报，提交 PR |
| 外贸政策监测 | 每日 07:00 UTC | 重外贸、海关、外汇、制裁来源，提交 PR |
| 地缘风险与外部环境观察 | 每日 07:30 UTC | 重台海、南海、中美、制裁清单，提交 PR |
| 政策蒸馏 | 每周一 07:30 UTC | 生成7种结构化卡片、周报，提交 PR |
| 技能维护 | 每月1日 08:00 UTC | 校准评分、审查推理框架，提交 PR |

### 你的 Agent 如何自动更新这个 Skill

本 Skill 定义在 `skill/SKILL.md` 中，支持**实时数据获取**——当你的 Agent（Hermes、opencode、Claude 等）加载此 Skill 后，它会：

1. **优先使用本地语料库回答** — `corpus/` 下的 55+ 份政策文档是即查即用的基础数据
2. **自动判断数据是否过时** — 当用户问"最新""最近"等问题，或本地语料覆盖不足时，Agent 会主动从官方来源实时抓取
3. **增量采集新文档** — 通过项目自带的采集脚本补充最新政策：

```bash
cd china-policy-analyze-skill
source venv/bin/activate
CPI_MAX_DOCS=5 python scripts/_run_daily_update.py
```

4. **拉取仓库最新更新** — 仓库的语料和报告每天自动更新，你的 Agent 只需 `git pull` 即可同步：

```bash
cd china-policy-analyze-skill && git pull origin main
```

> **提示：** 如果你的 Agent 支持 cron 定时任务（如 Hermes Agent），可以直接配置每天自动 `git pull` + 增量采集，实现完全无人值守的 Skill 更新。

## 核心理念

| 原则 | 说明 |
|------|------|
